# Pip Requirements for Skill Dockerfiles

This directory contains the source files used to generate the pinned Python dependencies and their secure hashes for the skill Dockerfiles.

These dependencies are **embedded** directly into the following Dockerfiles to ensure they are installed securely and verifiably without depending on the build context:
- `resources/Dockerfile.skill`
- `intrinsic_sdk_bundle_library_py/resource/skill.Dockerfile`

## Files

- `requirements.in`: Defines the top-level dependencies needed by the skills (e.g., `protobuf`, `grpcio`, `grpcio-status`).
- `requirements.txt`: The fully resolved dependency tree with pinned versions and SHA-256 hashes, autogenerated by `pip-compile`.
- `sync_requirements.py`: A utility script to synchronize the content of `requirements.txt` into the Dockerfiles.

## How to Update Dependencies

If you need to add, remove, or upgrade Python dependencies:

1. **Update the Input File:**
   Edit `requirements.in` to modify the top-level dependencies or pin them to specific versions if needed.

2. **Regenerate the Hashes:**
   Run `pip-compile` to regenerate `requirements.txt` with hashes. To ensure a clean environment and consistent hashes, it is recommended to run this via Docker:

   ```bash
   # Run from the root of the repository
   docker run --rm \
     -v $(pwd)/resources/pip_requirements:/apps \
     -w /apps \
     python:3.12 \
     bash -c "pip install pip-tools && pip-compile --generate-hashes requirements.in"
   ```

   This will update the `requirements.txt` file in this directory.

3. **Update the Dockerfiles:**
   Run the synchronization script to automatically update the embedded content in both Dockerfiles:

   ```bash
   ./resources/pip_requirements/sync_requirements.py
   ```

   *(Alternatively, you can manually copy the entire contents of `requirements.txt` and replace the embedded content in the here-doc section in `resources/Dockerfile.skill` and `intrinsic_sdk_bundle_library_py/resource/skill.Dockerfile`)*.

4. **Commit the Changes:**
   Commit the updated `requirements.in`, `requirements.txt`, the modified Dockerfiles, and any other changes.
